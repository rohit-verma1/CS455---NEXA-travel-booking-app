\section{\centering{\textbf{Architecture Design}}}

The overall architecture of our web application, Nexa, follows the Model-View-Controller (MVC) model. Given the complexity of the application, each functionality is subdivided into smaller tasks, which are achieved by leveraging the MVC architecture to ensure modularity, scalability, and maintainability.

The Model-View-Controller architecture is structured as follows:

\begin{itemize}
    \item \textbf{Model}: The model serves as the data-centric component of the application, acting as the sole component directly interfacing with the database. Its primary role entails storing and retrieving data related to user profiles, bookings, service provider schedules, payments, and analytics. We plan to use PostgreSQL as our database, with Django as the backend framework to interact with the database efficiently.
    
    \item \textbf{View}: The view acts as the presentation component of the application, responsible for rendering data to the user and handling user input. For customers and service providers, the view will be a responsive web application built using Ext.js ensuring a seamless and adaptive user interface across desktops, tablets, and smartphones. Administrators will access a dedicated web interface with enhanced controls for managing providers, bookings, and analytics.
    
    \item \textbf{Controller}: The controller serves as the logic hub of the application, tasked with processing data and determining the applicationâ€™s flow. It will be developed using ext.js with django to handle business logic, API integrations, and request routing. The integration of external APIs, such as Google Maps API for route visualization and real-time tracking, and a dummy payment gateway (e.g., Stripe sandbox) for payment simulations, will be managed here. AI-based features, such as recommendation systems and chatbots, will also be integrated through the controller.
\end{itemize}

We have opted for the Model-View-Controller (MVC) model because it effectively accommodates the diverse data interaction needs of Nexa. This model supports various operations, such as searching for travel options, managing bookings, handling dynamic pricing, and providing administrative oversight. The MVC architecture also facilitates future enhancements, such as adding multilingual support or integrating additional transportation modes, making it the best fit for our scalable and user-centric platform.
